Design a CareOps Clinic Administration Suite to run a rehab and long-term care facility’s daily operations entirely on a local network with an English-language interface. Front-desk and charge nurses use a fast, server-rendered HTMX UI to view beds, admissions, and occupancy at a glance, then drill into service delivery dashboards that show execution rate, work-order timeliness (on-time if completed within 15 minutes of the scheduled start), care-quality checkpoints, and alert events recorded by staff. Therapists and aides rely on an Exercise Library CMS to find techniques by tags, difficulty, body part, contraindications, and key coaching points, with mixed media stored locally; users can favorite items and the client caches recently viewed content with an LRU policy capped at 2 GB or 200 items, with a “Clear Device Cache” action for shared workstations. Training coordinators plan staff competency exams using reusable templates with subject, duration, and allowable time windows (e.g., 03/25/2026, 9:00 AM–12:00 PM), then generate sessions with conflict detection across rooms, proctors, and candidates and visually adjust time blocks before publishing. Finance clerks record payments and refunds for resident services using multiple offline “gateways” (cash, check, facility charge account, and imported card-terminal batches), perform shift-close settlement at 7:00 AM/3:00 PM/11:00 PM, and export reconciled financial reports to CSV/XLSX for handoff to accounting. Administrators and auditors can search audit trails, run permission-based reports with custom filters, schedule local report generation to a shared folder, and export datasets on demand.

The backend uses Go with Fiber to expose decoupled, REST-style endpoints consumed by HTMX partial updates, while SQLite persists all operational, clinical-content, scheduling, analytics, and finance records on the facility’s own hardware with no internet dependency. Key write paths enforce data consistency via optimistic locking (row version checks) and idempotency keys that expire after 24 hours to prevent duplicate submissions for create/update/approve/export actions; conflicts return a user-facing “record changed” prompt that reloads the latest state. Every privileged action appends an immutable audit log capturing operator identity, local IP, timestamp, and before/after change details, supporting traceability queries by resident, record type, or date range. Authentication is local username/password only, with minimum 12-character passwords, bcrypt hashing, role-based authorization, and automatic session timeout after 15 minutes of inactivity; sensitive fields (e.g., payer reference numbers) are encrypted at rest using AES-256-GCM with a locally stored key and masked in non-finance views. Observability is built-in with correlated request IDs, structured logs, job run history for scheduled reports and settlements, failure root-cause notes, configuration versioning with one-click rollback, and a one-click diagnostic package export that bundles logs, recent job results, and configuration snapshots into a local ZIP for offline troubleshooting.
